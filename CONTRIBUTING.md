# Contributing

This repository welcomes public-safe contributions that improve reusable methods
for AI video production control, Project Clinic, workflow rescue, and artifact
verification.

## Good Contributions

- synthetic examples that do not use private scripts, raw media, customer
  material, provider screenshots, account state, or final acceptance truth;
- clearer method language;
- blank templates that help non-engineer operators make a better first decision;
- issue reports about confusing wording, broken links, or missing public-safe
  examples;
- translations that preserve boundaries and non-claims.

## Good First Contribution Types

| Contribution | Done when |
| --- | --- |
| Filled synthetic example | The example uses invented material, links to a blank template, and names what it does not prove. |
| Template clarity fix | A non-engineer can fill the template without extra explanation. |
| Broken link or typo fix | The changed link resolves locally and on GitHub. |
| Public-safe method language | The wording helps a reader decide source, route, review, retake, or boundary. |

## Not Accepted

- private project assets;
- copied scripts, subtitles, screenshots, or copyrighted material;
- provider/account logs, keys, billing state, or job state;
- claims that a generated artifact is final or accepted by an owner;
- spam, bought stars, engagement farming, or fake social proof.

## How To Propose

1. Open the closest issue template.
2. Keep examples synthetic or self-owned.
3. Name what the method should help decide.
4. Name what the method does not prove.
5. Keep final acceptance with the actual project owner.
6. Use the pull request template if you submit a file change.
