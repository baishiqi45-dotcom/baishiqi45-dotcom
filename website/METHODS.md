# Public Method Library

These method cards are public-safe descriptions. They are designed to explain
judgment and workflow without exposing private project truth.

## Method 1: Five Continuity Locks For AI Video

Use when:

- a short-drama or AI video sequence keeps breaking character, space, prop,
  action, or emotional continuity;
- generated clips look individually acceptable but fail when edited together.

The five locks:

1. **Character lock**: what must remain recognizable across shots.
2. **Space lock**: what spatial facts cannot drift.
3. **Prop lock**: what object state must carry forward.
4. **Action lock**: what physical motion or causal action must continue.
5. **Emotional lock**: what performance state the next shot must inherit,
   refuse, or transform.

What it helps decide:

- whether a generated clip can enter edit review;
- which field needs retake;
- whether a failure is prompt, asset, sequence, or review design.

What it does not prove:

- market watchability;
- final edit readiness;
- copyright clearance;
- provider reliability.

## Method 2: Failure-Layer Debug Table

Use when:

- a team says "the AI output is bad" but cannot identify why;
- retake instructions keep becoming vague global prompts.

Failure layers:

| Layer | Question |
| --- | --- |
| Intent | Was the scene function clear? |
| Source | Did the references contain the needed evidence? |
| Asset | Did identity, prop, or space drift? |
| Prompt | Did the production instruction omit a hard constraint? |
| Generation | Did the provider fail despite clear constraints? |
| Review | Did the reviewer accept a weak artifact too early? |
| Edit | Did ordering or rhythm create the failure after generation? |

What it helps decide:

- where to retake;
- what not to touch;
- when to stop regenerating and rewrite the control packet.

What it does not prove:

- that one model is best;
- that all future scenes are solved;
- that the artifact is accepted by the owner.

## Method 3: DirectorShotIR In Plain Language

Use when:

- a creative team needs a director-like control language rather than a pretty
  prompt;
- multiple agents need separate responsibilities.

Core fields:

| Field | Plain-language meaning |
| --- | --- |
| Performance | What the actor/body should do and feel. |
| Blocking | Where bodies move and how they relate to space. |
| Camera | What the lens/viewer is responsible for noticing. |
| Transition | How this shot hands energy to the next shot. |
| Asset roles | Which visual references are binding and which are mood only. |
| Review rule | What would make this shot fail. |
| Retake target | The smallest field to change if it fails. |

What it helps decide:

- which agent owns which creative decision;
- how to avoid global "make it better" retakes;
- how to translate film judgment into bounded AI work.

What it does not prove:

- automated directing ability;
- final shot quality;
- real crew production readiness.

## Method 4: Project Clinic Startup Packet

Use when:

- a non-engineer has an AI product/workflow idea;
- the first step is unclear;
- the project risks jumping into tools before source and acceptance are known.

Startup packet:

```text
Original intent
Known sources
Missing sources
Candidate tool route
Owner truth boundary
First safe action
Review checklist
Outcome-delta note
```

What it helps decide:

- whether to research, prototype, hire, prompt, or stop;
- what an agent should read first;
- what evidence would change the next action.

What it does not prove:

- product-market fit;
- runtime readiness;
- engineering completeness.

## Method 5: Video Admission Ladder

Use when:

- video artifacts exist but the team does not know whether they can enter edit
  or story judgment;
- model summaries or sidecar reviews are being treated as proof.

Admission ladder:

1. **Source evidence**: what raw artifact exists.
2. **Static evidence**: frames, contact sheets, manifest, timestamps.
3. **Playback evidence**: human watched the relevant span.
4. **Relation evidence**: what this clip must connect to before/after.
5. **Owner seal**: who can call it accepted for the project.

What it helps decide:

- whether a clip is reviewable;
- whether a timeline is only candidate;
- whether a continuity claim is too strong.

What it does not prove:

- automated video understanding;
- final edit acceptance;
- commercial delivery.
